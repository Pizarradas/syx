Architecture

How Syx works

A three-layer token system that separates raw values from meaning, and meaning from usage.

1

Primitive Tokens

Raw values — colors, spacing, type scales. The single source of truth.

2

Semantic Tokens

Contextual aliases — --semantic-color-primary maps to the right primitive per theme.

3

Component Tokens

Scoped decisions — each component consumes semantic tokens, never primitives.

Core Principles

Built for the modern web

Three pillars that make Syx different from any other design system.

Atomic Architecture

Built on the Atomic Design methodology. Atoms, molecules, and organisms work together to build complex UIs with zero redundancy.

Semantic Tokens

Theming made easy. Switch between brands or modes (Dark/Light) purely by changing token mappings — no component changes needed.

Fluid Typography

Typography that scales organically with the viewport using CSS clamp(). No more jerky breakpoints — just smooth, continuous scaling.

Accessibility First

WCAG 2.1 AA compliant focus states, ARIA patterns, and reduced-motion support baked into every component.

Multi-Theme Ready

Ship multiple brand themes from a single codebase. Each theme is a CSS class override — zero duplication.

Developer Experience

SCSS mixins, utility classes, and a clear naming convention make Syx fast to learn and a joy to use every day.

Mobile-First

Every component is designed mobile-first. Named breakpoints (phablet, tablet, desktop) and @include breakpoint() make responsive design systematic, not an afterthought.
