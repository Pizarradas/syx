Architecture

How Syx works

A three-layer token system that separates raw values from meaning, and meaning from usage.

01

Primitive Tokens

Raw values — colors, spacing, type scales. The single source of truth shared across every theme.

--primitive-color-blue-500
02

Semantic Tokens

Contextual aliases — --semantic-color-primary maps the right primitive per active theme.

--semantic-color-primary
03

Component Tokens

Scoped decisions — each component consumes semantic tokens, never primitives. Isolated, predictable, themeable.

--component-btn-bg
Core Principles

Built for the modern web

Three pillars that make Syx different from any other design system.

Atomic Architecture

Built on Atomic Design. Atoms, molecules, and organisms work together to build complex UIs with zero redundancy.

Semantic Tokens

Switch brands or modes (Dark/Light) purely by changing token mappings — no component changes needed.

Fluid Typography

Typography that scales organically with the viewport using CSS clamp(). No more jerky breakpoints.
