Architecture

SYX uses CSS @layer to manage specificity without !important. Utilities always win over components — by design.

CSS @layer Stack

@layer syx.reset, syx.base, syx.tokens, syx.atoms, syx.molecules, syx.organisms, syx.utilities;
CSS layer stack — order and precedence
Layer Content Wins over
syx.reset Browser reset —
syx.base Element defaults, helpers reset
syx.tokens CSS custom property tokens base
syx.atoms Atomic components tokens
syx.molecules Composite components atoms
syx.organisms Complex UI sections molecules
syx.utilities Utility classes everything

Token Architecture

Primitive → Semantic → Component

Never use primitive tokens directly in components. Always go through semantic → component.

Token architecture layers
Layer Example Rule
Primitive --primitive-color-purple-500 Raw values, no context
Semantic --semantic-color-primary Contextual aliases
Component --component-btn-primary-bg Component-specific

Iconography

Icons are inline SVGs stored as CSS custom properties (data URIs). Use the .syx-icon atom with BEM modifiers — no inline styles needed.

Token Architecture

Layer 1 — Primitive: --primitive-icon-* → stores the raw SVG data URI
Layer 2 — Semantic: --icon-* → alias consumed by components
Colorization: filter: var(--semantic-filter-*) → recolors the black SVG to any brand color

All icons are black SVGs. Add a --color-* modifier to recolor via CSS filter.

--color-state-ko

Usage in HTML

<!-- Basic icon -->
<span class="syx-icon syx-icon--arrow-default" aria-hidden="true"></span>

<!-- With size modifier -->
<span class="syx-icon syx-icon--ui-home syx-icon--lg" aria-hidden="true"></span>

<!-- With color modifier -->
<span class="syx-icon syx-icon--ui-tick syx-icon--md syx-icon--color-state-ok" aria-hidden="true"></span>

<!-- With accessible label (when icon conveys meaning) -->
<span class="syx-icon syx-icon--ui-cancel syx-icon--md" role="img" aria-label="Close"></span>
