How to Style with SYX

Practical reference for every mixin, token pattern, and component recipe. Each section shows the SCSS you write and the CSS it generates.

Token System

SYX uses 3 token layers. Always consume the most specific layer available.

Rule: Components use --component-* tokens. Themes use --semantic-*. Never use --primitive-* directly in components.

Layer 1  Primitive (raw values)

// ? Don't use in components color: var(--primitive-color-purple-500); font-size: var(--primitive-font-size-16);

Layer 2  Semantic (contextual aliases)

// ? Use in theme-level overrides and layout color: var(--semantic-color-primary); background: var(--semantic-color-surface-secondary); border-radius: var(--semantic-border-radius-md);

Layer 3  Component (per-component)

// ? Use inside component mixins color: var(--component-btn-primary-color); background: var(--component-btn-primary-bg); border-radius: var(--component-btn-border-radius);

Semantic color tokens

--semantic-color-primary Main brand color
--semantic-color-surface-secondary Page background alt
--semantic-color-border Default border
--semantic-color-text-primary Body text

Positioning Mixins

Null-safe shorthands. Pass null to skip any side  no empty CSS emitted.

Null-safety: @include absolute($top: 0) emits only position: absolute; top: 0;  not right: null.

@include absolute()

SCSS
// Full overlay @include absolute($top: 0, $right: 0, $bottom: 0, $left: 0); // Top-right corner @include absolute($top: 1rem, $right: 1rem); // Centered horizontally at bottom @include absolute($bottom: 0, $left: 50%);
Generated CSS
position: absolute; top: 0; right: 0; bottom: 0; left: 0; position: absolute; top: 1rem; right: 1rem; position: absolute; bottom: 0; left: 50%;

@include sticky()

SCSS
// Sticky header .site-header { @include sticky($top: 0); z-index: 100; }
Generated CSS
position: sticky; top: 0; z-index: 100;

@include relative() / @include fixed()

// Positioning context for children .card { @include relative(); } // Fixed bottom bar .cookie-bar { @include fixed($bottom: 0, $left: 0, $right: 0); }

Spacing Mixins

@include margin() and @include padding() follow the CSS shorthand order (top right bottom left). Pass null to skip a side.

@include margin()

SCSS ? CSS
@include margin(null auto); // ? margin-left: auto; margin-right: auto; @include margin(2rem null 1rem null); // ? margin-top: 2rem; margin-bottom: 1rem; @include margin(1rem); // ? margin: 1rem; // With tokens @include margin(var(--layout-gap-4) null); // ? margin-top: var(--layout-gap-4); margin-bottom: var(--layout-gap-4);

@include padding()

@include padding(1rem null); // ? padding-top: 1rem; padding-bottom: 1rem; @include padding(var(--component-card-padding-y) var(--component-card-padding-x)); // ? padding-top: ...; padding-bottom: ...; padding-left: ...; padding-right: ...; // Skip left/right (vertical only) @include padding(var(--layout-pad-3) null); // ? padding-top: var(--layout-pad-3); padding-bottom: var(--layout-pad-3);
Tip: For symmetric padding use @include padding(Y X)  2 values. For asymmetric use all 4: @include padding(top right bottom left).

Sizing Mixin

@include size() sets width and height. One argument = square.

SCSS ? CSS
// Square (one arg) @include size(48px); // ? width: 48px; height: 48px; // Rectangle (two args) @include size(100%, 56px); // ? width: 100%; height: 56px; // With tokens @include size(var(--component-avatar-size)); // ? width: var(--component-avatar-size); height: var(--component-avatar-size);

@include aspect-ratio()

// 16:9 video embed .video-wrapper { @include aspect-ratio(16, 9); } // Square thumbnail .thumb { @include aspect-ratio(1, 1); } // Portrait card .portrait { @include aspect-ratio(3, 4); }
16 / 9 aspect ratio

Text Mixins

@include truncate()  single line

// Truncate to max-width (default: 100%) .card__title { @include truncate(240px); } // Truncate to container width .list-item__label { @include truncate(); }

Animation Mixin

@include transition() automatically adds a prefers-reduced-motion guard Â no extra code needed.

SCSS
// Single property .atom-btn { @include transition(background-color 0.2s ease); } // Multiple properties .mol-card { @include transition( transform 0.3s cubic-bezier(0.25, 0.8, 0.25, 1), box-shadow 0.3s ease ); }
Generated CSS
/* Always emits the reduced-motion guard automatically */ .atom-btn { transition: background-color 0.2s ease; } @media (prefers-reduced-motion: reduce) { .atom-btn { transition: none; } }
? Best practice: Always use @include transition() instead of bare transition:. The reduced-motion guard is free.

Media Query Mixins

Mobile-first (min-width) named breakpoints. Em-based for zoom safety.

Breakpoint scale

SYX breakpoint scale
Name Direction Value Pixels (approx)
mobile-only max-width 37.5em = 600px
phablet min-width 37.5em = 600px
tablet min-width 50em = 800px
laptop min-width 64em = 1024px
desktop min-width 70em = 1120px
wide min-width 90em = 1440px

@include breakpoint()

SCSS
// Mobile-first: style mobile, override upward .hero__title { font-size: 2rem; // mobile default @include breakpoint(tablet) { font-size: 3rem; // = 800px } @include breakpoint(desktop) { font-size: 4rem; // = 1120px } } // Mobile-only exception .mobile-menu { display: block; @include breakpoint(tablet) { display: none; } }
Generated CSS
.hero__title { font-size: 2rem; } @media (min-width: 50em) { .hero__title { font-size: 3rem; } } @media (min-width: 70em) { .hero__title { font-size: 4rem; } }

@include darkmode

.card { background: var(--semantic-color-surface); @include darkmode { background: var(--semantic-color-surface-dark, #1a1a2e); } }

@include reduced-motion

// Manual override (usually not needed  @include transition() handles this) .animated-element { animation: slide-in 0.4s ease; @include reduced-motion { animation: none; } }

Accessibility Mixins

@include sr-only()

Visually hides an element while keeping it accessible to screen readers.

// Skip navigation link .skip-link { @include sr-only(); &:focus { @include sr-only-reset(); @include absolute($top: 0, $left: 0); z-index: 9999; } } // Icon button label .icon-btn__label { @include sr-only(); }

@include focus-ring()

WCAG-compliant focus outline. Uses outline (not box-shadow) so it's not clipped by overflow: hidden.

// Default (uses --color-primary) a:focus-visible { @include focus-ring(); } // Custom color and offset .custom-input:focus { @include focus-ring($color: var(--semantic-color-primary), $offset: 4px); }

Component Recipes

Real-world patterns combining multiple mixins and tokens.

Recipe 1  Overlay badge

// A badge positioned top-right of a card .card { @include relative(); // positioning context } .card__badge { @include absolute($top: -0.5rem, $right: -0.5rem); @include size(1.5rem); @include flex-center(); border-radius: 50%; background: var(--semantic-color-primary); color: #fff; font-size: 0.75rem; font-weight: 700; }

Recipe 2  Responsive card grid

// 1 col mobile ? 2 col tablet ? 3 col desktop .card-grid { display: grid; grid-template-columns: 1fr; gap: var(--layout-gap-4); @include breakpoint(tablet) { grid-template-columns: repeat(2, 1fr); } @include breakpoint(desktop) { grid-template-columns: repeat(3, 1fr); } } .card-grid__item { @include transition(transform 0.2s ease, box-shadow 0.2s ease); &:hover { transform: translateY(-4px); box-shadow: 0 8px 24px rgba(0,0,0,0.1); } }

Recipe 3  New atom from scratch

// scss/atoms/_tag.scss @use "../abstracts/index" as *; @mixin atom-tag($theme: null) { .syx-tag { display: inline-flex; align-items: center; gap: 0.25rem; @include padding(0.25rem 0.625rem); border-radius: var(--component-tag-border-radius, var(--semantic-border-radius-sm)); background: var(--component-tag-bg, var(--semantic-color-surface-secondary)); color: var(--component-tag-color, var(--semantic-color-text-primary)); font-size: var(--semantic-font-size-body-small); font-weight: 500; @include transition(background-color 0.15s ease); &--primary { background: var(--component-tag-primary-bg, var(--semantic-color-primary)); color: var(--component-tag-primary-color, #fff); } } }

Anti-patterns

Common mistakes and how to fix them.

? Inline position instead of mixin

// ? Wrong .card { position: relative; } .badge { position: absolute; top: 0; right: 0; }
// ? Correct .card { @include relative(); } .badge { @include absolute($top: 0, $right: 0); }

? Inline transition instead of mixin

// ? Wrong Â no reduced-motion guard .btn { transition: background-color 0.2s ease; }
// ? Correct Â guard included automatically .btn { @include transition(background-color 0.2s ease); }

? Hardcoded values instead of tokens

// ? Wrong .card { border-radius: 8px; color: #1a1a2e; }
// ? Correct .card { border-radius: var(--semantic-border-radius-md); color: var(--semantic-color-text-primary); }

? Using !important

// ? Wrong Â @layer makes this unnecessary .syx-d-none { display: none !important; }
// ? Correct Â @layer syx.utilities wins by position @layer syx.utilities { .syx-d-none { display: none; } }

? Using primitive tokens in components

// ? Wrong Â breaks theming .atom-btn { background: var(--primitive-color-purple-500); }
// ? Correct Â theme-swappable .atom-btn { background: var(--component-btn-primary-bg); }
Rule of thumb: If you find yourself writing position:, transition:, !important, or a hex color directly in a component Â stop and use the mixin or token equivalent.

Single-Partial Multi-Theme Pattern

One SCSS file per component handles all themes via three distinct mechanisms. No per-theme copies, no duplication.

Method 1 — CSS Custom Property var()

For values that all themes have but differ in value (colors, spacing, icons). Use a generic token in the partial; each _theme.scss overrides it.

// organisms/_header.scss background-color: var(--component-header-bg); // generic â every theme has this background-image: var(--component-header-logo-icon); // generic â every theme overrides it // themes/example-02/_theme.scss (Codymer) --component-header-logo-icon: var(--icon-logo-codymer); // themes/example-03/_theme.scss (Coral) --component-header-logo-icon: var(--icon-logo-coral);

Method 2 — Sass Map theme-cfg()

For structural differences resolved at compile time (layout, position, Sass-only sizes). Values live in abstracts/_theme-config.scss and are read with theme-cfg().

// abstracts/_theme-config.scss $theme-config: ( 'codymer': (header-sidenav-side: right, header-logo-size: 2.4rem), 'coral': (header-sidenav-side: left, header-logo-size: 2rem), ); // organisms/_header.scss â reads the map at compile time @if theme-cfg($theme, 'header-sidenav-side', left) == right { right: 0; transform: translateX(100%); // Codymer: slides in from right } @else { left: 0; transform: translateX(-100%); // all others: slides in from left }

Method 3 — @if $theme

For one-off rules that only 1-2 themes need. No token or map entry required — inline override in the partial.

// organisms/_header.scss @if $theme == 'codymer' { border-bottom: 1px solid var(--semantic-color-border-subtle); // Codymer only } @if $theme == 'midnight' { backdrop-filter: blur(8px); // Midnight glass effect only }

Decision table

Question Method
Do all themes need this, but with different values? Method 1 — CSS token
Is it structural and reused in multiple spots of the mixin? Method 2 — Sass Map
Is it specific to only 1-2 themes and not worth tokenizing? Method 3 — @if $theme
Adding a new theme variant: add an entry to $theme-config, define token overrides in _theme.scss, pass the name to the mixin. No new SCSS partials ever needed.
