Utilities

Atomic utility classes for handling layout, spacing, and typography directly in HTML. Format: .syx-{property}-{value}.

Why No !important?

SYX utilities live in the syx.utilities CSS layer Â the highest in the stack. This means they always override component styles without needing !important.

/* SYX @layer stack */ @layer syx.reset, syx.base, syx.tokens, syx.atoms, syx.molecules, syx.organisms, syx.utilities; /* Utilities win by position, not by specificity */ @layer syx.utilities { .syx-d-none { display: none; } /* always wins over .atom-btn */ }

Rule: If a utility isn't overriding a component, check that both are compiled in the correct layer. Never add !important Â it's a bug.
